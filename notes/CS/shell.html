1 cheat sheet

1.1 cat

  • 编辑新文件：
    cat > some_file

1.2 sftp

  • 连接：sftp <username>@<ip address>

  • 文件传输：

    • 上传：get -r <remote-path> <local-path>
    • 下载：put -r <local-path> <remote-path>

  • 命令操作时，默认是远端的环境，如果要在本地环境下执行命令，则所有命令加上前缀“l”，如lls

1.3 dpkg

  • 查看所有安装包：dpkg -l
  • 查看某个安装包所有的相关文件：dpkg -L <package-name>
  • 查看某个deb包内部的文件：dpgk -c <deb-name.deb>
  • 卸载某个包（及其配置文件）：dpkg -P <package-name>
  • 下載dpkg包：apt download <package-name>

1.4 sed

  • 替换文件的某一行：sed -i 's/<old-pattern>/<new-pattern>/g' <filename>

1.5 echo

  • 输出有颜色的文本： echo -e "\\e\[1;<color-code>m<output string>\\e\[0m" ，其中 color-code=31 表示红色， color-code=32 为绿色；

1.6 read

用于读取用户的输入。

  • 读入用户的输入：read -p "Input your string: " $YOUR_STRING
  • 读入密码：read -s -p "Password: " $YOUR_PASSWORD

1.7 losetup

用于管理loop设备。

  • 将文件转换为块设备：losetup /dev/loop0 <path-to-image>
  • 删除loop设备：losetup -d /dev/loop0
  • 查看所有正在使用的loop设备：losetup -a

1.8 mount

  • 挂在iso：mount <path-to-ISO> <mount-dir> -o loop

1.9 xargs

  • 指定参数：find . -name "**\*.html" | xargs -i echo {}或者find . -name "\***.html" | xargs -I {} echo {}

1.10 iptables

基本语法：

  • 查看规则：iptables -nvL –line-number
  • 添加规则：
    • 添加到末尾：iptables -A INPUT -s 192.168.1.5 -j DROP
    • 添加到第三行：iptables -I INPUT 3 -s 192.168.1.5 -j DROP
  • 删除规则：iptables -D INPUT -s 192.168.1.5 -j DROP或者iptables -D INPUT 2
  • 修改规则：iptables -R INPUT 3 -j ACCEPT

Cheat Sheet：

  • 开放本机的80端口：iptables -A INPUT -p tcp –dport 80 -j ACCEPT

1.11 alias

如果需要临时关闭alias：unalias rm

1.12 basename

获取一个路径中的文件名部分，如果文件名有扩展“filename.ext”，通过 basename /path/to/filename.ext .ext可以获取“filename”。

1.13 mysql

  • 备份database：mysqldump -uxx -pxx –all-databases > databases.sql
  • 授权：grant select on testdb. to common_user@'%';
  • 撤销授权：revoke all on *.* from dba@localhost;
  • 查看授权：show grants for dba@localhost;

1.14 iostat

检测每个硬盘的IO情况。

iostat -x 2 5：每2秒打印一次报告，共打印5次。

1.15 ps

  • 打印出处于uninterruptible sleep状态（通常就是处于IO wait）的进程：
    for x in `seq 1 1 10`; do ps -eo state,pid,cmd | grep "^D"; echo "----"; sleep 5; done
  • 查看进程的已运行时间：ps -p 24525 -o lstart,etime
  • 查看内存占用前10的进程：ps -aux | sort -k4nr | head -10

1.16 lsof

  • 查看进程读写状况：lsof -p <pid>

1.17 pvdisplay, vgdisplay, lvdisplay

查看虚拟卷组情况。

1.18 find

  • 查看最近5天编辑过的 C 文件：find -name "*.c" -mtime -5
  • 查找符合条件的文件并执行指定的命令：find -name "*.md" -exec file {} \;，这其中的\;是作为命令终结的标志。

1.19 man

  • 想要某个功能但是不知道使用哪个命令可以使用man自带的搜索功能：man -k "copy files"

1.20 size

  • 查看可执行程序中各个区间（text 段、data 段、bss 段等）的大小：size /bin/ls

1.21 objdump

  • 简单的反汇编程序可以直接objdump -d /bin/ls
  • 如果已经知道了某一个函数，只想知道该函数的汇编代码：objdump -d /bin/ls --start-address=0x40052d --stop-address=0x400540，其中地址可以通过nm查看。

1.22 netstat

  • 查看端口占用：netstat -anp | grep <some port>，ss -nlp | grep <some port>也可。

1.23 addr2line

  • 程序崩溃但是没有生成 core 文件，通过 dmsg 可以得知程序出错的基本原因和出错位置，使用addr2line可以获取出错的具体行号：addr2line -e cmdTest 40053b

2 troubleshooting

2.1 Slowness

首先根据top中的iowait来进行排查是否是IO导致，然后再使用iotop查看哪个进程的IO较高， 结合lsof -p <pid>可以查看该进程写入的文件，根据文件的位置并结合iostat和pvdisplay的结果来查看该文件是否在高IO的磁盘上1。

如果没有iotop命令，可以直接使用ps定位：

    for x in `seq 1 1 10`; do ps -eo state,pid,cmd | grep "^D"; echo "----"; sleep 5; done

提供了查看在uninterruptible sleep状态的进程（通常是由于IO）， 结合查看/proc系统中的信息 cat /proc/<pid>/io可以确定进程的IO大小， 接下来可以结合lsof，iostat等确定进程是否 是高IO。
